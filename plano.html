REESTRUTURAÇÃO DO SITE

everybody needs a brah — but the site needs work.

Resumo do plano para o redesign do brah-site, baseado nas notas da reunião com o cliente e nas 3 referências (Vício, Street Smash, Honest Greens). Acompanha o protótipo clicável onde podes comparar as 3 direções lado a lado.

1. O que vamos arrumar

Notas do cliente, traduzidas em decisões:

tirar preçosO menu deixa de ter preços. Cada burger fica com nome + ingredientes + tag (the og, best seller, loaded, vegan). CTA "ask in store" ou "see in store".

3. Sistema visual aplicado

Tudo enraizado no brah Design System que já existe. Cores, fontes, espaçamento, todos via tokens (--brah-tomato, --font-display, etc.).

  • Tipografia — Lastik (display, todos os headlines), Work Sans (corpo), Toms Handwritten (notas de margem, abertura "open everyday"). Apenas uma display font por superfície.
  • Cores — Cream (#FFFCE4) como background base. Black para texto. Tomato (#F26B44) para stamps e CTA. Sky / Lavender / Peach apenas como acentos.
  • Layout — Cantos a 90° (sem rounded corners), exceto stickers (18px). 1px black hairline entre secções. Sem sombras suaves, sem glassmorphism.
  • Tom de copy — segunda pessoa, em lowercase. "brah" sempre minúsculo. Headlines em EN; informação prática em PT (com toggle global). Aspas curvas "…".
  • Iconografia — quase nenhum. Sun glyph como acento ocasional. Sem emoji.

4. Três direções (no protótipo)

DIRECTION A · Vício

Confident, burger-first.

Tipografia gigante, foto de produto a ocupar metade do hero. Lista vertical de burgers com fade entre fotos no hover. Cream + black + tomato. Menu cinematográfico em alternado esquerda/direita. Layout limpo, sem chaos.
Best for: público adulto, primeira impressão de marca séria.

DIRECTION B · Street Smash

Poster, raw, evento.

Sky/lavender backgrounds, stamps por todo o lado, stickers rodados, sun glyph a girar, sombras hard offset. Eventos com indigo + tomato. Galeria collage. Mais "energia de cartaz" que website.
Best for: falar com surfistas, jovens, pessoal dos pop-ups.

DIRECTION C · BLEND

Híbrido — recomendado.

Base limpa do Vício com acentos do Street Smash: stamps, notas escritas à mão coloridas, sombras hard-offset nos cards de produto, galeria com fotos rodadas. A página de Eventos vai em modo Street Smash completo (indigo + tomato). Mantém a marca séria onde conta, e solta o lado poster onde faz sentido.

EXTRAS · em todas as 3 direções

Reviews · Story · Cinematic menu.

Adicionei a todas: secção Reviews mock (Google / TripAdvisor / Instagram), página de Story enriquecida (cronologia, pull quote), menu cinematográfico (alternado, fig. captions), microinterações (hover lifts, button press, focus rings tomato), e mobile responsive.

5. O que muda em concreto vs. o site atual

  1. Remover a animação do hambúrguer a abrir (burger-scroll.js). Substituir por foto estática no hero.
  2. Remover todos os preços do menu. Manter ingredientes e tags.
  3. Adicionar 4 páginas novas: events.html, gallery.html, catering.html, story.html (se ainda não tiveres).
  4. Adicionar toggle PT/EN global (estado guardado no localStorage).
  5. Adicionar meta strip no topo (morada · horário · IG · FB).
  6. Adicionar secção dedicada de Instagram (6 tiles, grid 6×1) em todas as páginas, antes do footer.
  7. Reforçar Instagram e Facebook em 3 sítios: meta strip + header + footer.
  8. Simplificar o header — só wordmark + 7 links + 1 CTA. Sem submenus, sem dropdowns.
  9. Migrar os tokens do design system (colors_and_type.css) e usar var(--brah-*) em todos os componentes.

6. Próximos passos

  1. Hoje: abrir o protótipo, alternar entre as 3 direções, decidir qual (ou mistura).
  2. Esta semana: validar conteúdo final (ingredientes corretos? eventos confirmados? fotos definitivas?).
  3. Próxima sprint: implementar a direção escolhida no repo brah-site. Fazer commit dos novos ficheiros. Atualizar index.html, styles.css, criar as 4 páginas novas.
  4. Antes de live: meta-tags OG (Facebook share), favicon, Google Analytics, sitemap. Testar em mobile.
Notas técnicas para o Claude Code / dev:
— Os ficheiros do protótipo (src/*.jsx) já estão modulares: cada direção é um ficheiro independente. Podem ser portados quase 1:1 para componentes do repo.
— Os tokens estão em css/tokens.css + css/kit.css, prontos para copiar para o repo.
— Os assets (fotos, logos, fontes) estão em assets/ e fonts/, já com nomes consistentes.
— O toggle PT/EN é puro React state — para o site de produção, considerar guardar em localStorage ou usar URL params (?lang=pt).
